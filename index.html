Airflow DAG — Graph View

Four tasks with explicit dependency management. Both extract tasks run in parallel. Transform waits for both. Load waits for transform. All runs write to a single unified pipeline.log.

About the @daily schedule: When running locally, Airflow requires both the webserver (airflow webserver --port 8080) and scheduler (airflow scheduler) to be running simultaneously in separate terminals. The @daily schedule triggers automatically once both are active — the scheduler watches for the next scheduled interval and executes the DAG without manual intervention. The pipeline can also be triggered manually at any time via the ▶ button in the UI.

Real anomalies on July 2: ethereum and solana both flagged — their prices had moved more than 5% vs. the previous stored values from June 30. The pipeline preserved both rows with is_anomaly=True rather than silently dropping them, exactly as designed for financial data systems.
Why FX rates look identical July 3–6: The ECB (European Central Bank) publishes exchange rates once per day at ~15:00 UTC. Runs on July 3 (16:57 UTC) got fresh rates. All subsequent runs on July 4, 5, 6 executed before 15:00 UTC, so Frankfurter returned the same July 3 publication. This is correct API behaviour — the pipeline faithfully records what was available at fetch time. Crypto prices (bitcoin, ethereum, solana) change continuously and show genuine movement every day since CoinGecko updates in real time.
